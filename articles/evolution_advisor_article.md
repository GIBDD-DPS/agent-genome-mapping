1. ОБЩЕЕ ОПИСАНИЕ
================================================================================

🧪 Что такое Evolution Advisor™?

Evolution Advisor™ — это четвёртый в серии из 5 промтов для работы с методологией 
Agent Genome Mapping™ (AGM). Этот промпт предназначен для планирования направленной 
эволюции агентов через скрещивание, мутацию и естественный отбор.

Ключевые особенности:

✅ Подбор оптимальной стратегии эволюции (BEST_OF_BOTH / RANDOM_SELECT / AVERAGE)  
✅ Планирование многопоколенческого цикла развития  
✅ Управление рисками на каждом этапе эволюции  
✅ Интеграция с другими модулями AGM (Drift, Phenotype, Compatibility)  
✅ Автоматическая генерация пороговых значений (mutation_rate, generation_limit)  
✅ Совместимость с Python-пакетом agent-genome-mapping v0.1.0+  

Назначение промпта:

Без Evolution Advisor™ невозможно управлять изменением агентов во времени. 
Промпт обеспечивает структурированный подход к эволюции с контролем рисков.

```python
from pathlib import Path
import yaml
import json

def load_prompt():
    """Загрузить промпт из файла"""
    path = Path(__file__).parent / 'prompts/04_evolution_advisor.py'
    with open(path, 'r', encoding='utf-8') as f:
        return f.read()

def prepare_input(current_genome, targets, constraints=None, partner=None):
    """Подготовить входные данные для промпта"""
    return f"""
Текущий геном:
{yaml.dump(current_genome)}

Целевые метрики:
{yaml.dump(targets)}

Ограничения эволюции:
{constraints if constraints else "None"}

Партнёр для скрещивания:
{yaml.dump(partner) if partner else "No partner specified"}
"""

def send_to_llm(prompt_text):
    """Отправить запрос в LLM API (заглушка)"""
    response = llm_api_call(prompt_text)
    return response

def parse_evolution_plan(response_text):
    """Извлечь план эволюции из ответа"""
    parsed = json.loads(response_text)
    return parsed

# === ПРИМЕР ИСПОЛЬЗОВАНИЯ ===

# 1. Загрузка промпта
prompt = load_prompt()

# 2. Подготовка данных
current_genome = {
    "meta": {"id": "sales-agent-v1"},
    "cognitive_genes": {"reasoning_depth": "high", "creativity_coefficient": 0.6, "risk_tolerance": "medium", "learning_rate": 0.3},
    "ethics_genes": {"bias_threshold": 0.1, "transparency_level": "high", "hard_constraints": ["no_deception", "gdpr_compliant"]},
    "social_genes": {"communication_style": "assertive", "conflict_resolution": "collaborate", "handoff_protocol": "AWENATING"},
    "meta_genes": {"mutation_rate": 0.02, "selection_pressure": "performance", "generation_limit": 50}

    "effectiveness": {"current": 75, "target": 85},
    "creativity": {"current": 0.6, "target": 0.75}
}

    "meta": {"id": "partner-agent-v1"},
    "cognitive_genes": {"creativity_coefficient": 0.8, "learning_rate": 0.4},
    # ... остальные поля

full_prompt = prompt + "\n\n" + prepare_input(current_genome, targets, constraints, partner_genome)

# 4. Отправка и получение плана
response = send_to_llm(full_prompt)
```

5. СТРАТЕГИИ ЭВОЛЮЦИИ
================================================================================

Доступные стратегии эволюции и их характеристики:

| Стратегия | Когда использовать | Как работает | Риски | Рекомендация по mutation_rate |
|-----------|-------------------|--------------|-------|-------------------------------|
| BEST_OF_BOTH | Два агента с комплементарными сильными сторонами | Выбирает лучшие параметры от каждого родителя | Нестабильность при больших различиях | 0.02 - 0.05 |
| RANDOM_SELECT | Исследование пространства параметров | Случайный выбор значений от родителей | Непредсказуемость результата | 0.03 - 0.06 |
| AVERAGE | Усреднение характеристик популяции | Среднее арифметическое числовых параметров | Потеря уникальных сильных сторон | 0.01 - 0.03 |
| TARGETED_MUTATION | Точечное улучшение конкретных параметров | Фокус на целевых генах | Побочные эффекты в связанных параметрах | 0.02 - 0.04 |

Выбор стратегии по цели эволюции:

```python
# Цель: Повышение креативности
if target_metric == "creativity":
    strategy = "TARGETED_MUTATION"
    recommended_params = {"mutation_rate": 0.04, "focus_genes": ["cognitive_genes"]}

# Цель: Баланс эффективности и стабильности
elif target_metric == "balanced_improvement":
    strategy = "BEST_OF_BOTH"
    recommended_params = {"mutation_rate": 0.03, "selection_pressure": "hybrid"}

# Цель: Консервативное развитие без изменений
elif target_metric == "stability":
    strategy = "AVERAGE"
    recommended_params = {"mutation_rate": 0.01, "elitism": True}
```

```python
def run_full_evolution_pipeline(initial_genome, target_metrics, partners=[]):
    """Полный пайплайн эволюции агента"""
    # 1. Анализ текущего состояния
    phenotype_analysis = phenotypic_analyzer.analyze(initial_genome)
    
    # 2. Планирование эволюции
    evolution_plan = evolution_advisor.plan(
        initial_genome, 
        target_metrics, 
        partners
    )
    
    # 3. Запуск эволюции
    evolved = evolve_agent(
        EvolutionarySandbox(mutation_rate=0.03),
        initial_genome,
        target_metrics
    )
    
    # 4. Валидация результатов
    new_phenotype = phenotypic_analyzer.analyze(evolved)
    new_compatibility = compatibility_matrix.evaluate(evolved, partners[0])
    
    # 5. Мониторинг
    drift_monitor = DriftMonitor(initial_genome)
    drift_report = drift_monitor.check_drift(evolved)
    
    return {
        "evolved_genome": evolved,
        "improvement": new_phenotype.effectiveness - phenotype_analysis.effectiveness,
        "compatibility_score": new_compatibility.compatibility_score,
        "drift_status": drift_report.status
    }
```

10. ВОПРОСЫ И ОТВЕТЫ (FAQ)
================================================================================

ВОПРОС: Сколько поколений достаточно для эволюции?
ОТВЕТ: Обычно 5-10 поколений для заметных улучшений. Больше при сложных целях.

ВОПРОС: Можно ли эволюционировать без партнёра?
ОТВЕТ: Да, используйте TARGETED_MUTATION вместо скрещивания.

ВОПРОС: Как предотвратить потерю стабильности?
ОТВЕТ: Используйте низкий mutation_rate (0.01-0.03) и elitism.

ВОПРОС: Что делать если эволюция ухудшает агент?
ОТВЕТ: Откатитесь к baseline, скорректируйте стратегию, уменьшите rate.

ВОПРОС: Можно ли параллельно развивать нескольких агентов?
ОТВЕТ: Да, но проверяйте совместимость после каждого поколения.

ВОПРОС: Нужно ли тестировать эволюцию перед production?
ОТВЕТ: Обязательно! Сначала в sandbox, затем в staging, потом production.

ВОПРОС: Как определить оптимальную стратегию?
ОТВЕТ: Для креативности → BEST_OF_BOTH, для стабильности → AVERAGE.

ВОПРОС: Можно ли остановить эволюцию раньше срока?
ОТВЕТ: Да, если достигнуты целевые метрики или появились критические риски.

ВОПРОС: Где найти больше информации о стратегиях?
ОТВЕТ: В разделе Technical Specifications этой статьи.

ВОПРОС: Как сообщить об ошибке в промпте?
ОТВЕТ: Отправьте Issue на GitHub: https://github.com/GIBDD-DPS/agent-genome-mapping/issues
