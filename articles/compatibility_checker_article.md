1. ОБЩЕЕ ОПИСАНИЕ
================================================================================

🔗 Что такое Compatibility Checker™?

Compatibility Checker™ — это третий в серии из 5 промтов для работы с методологией 
Agent Genome Mapping™ (AGM). Этот промпт предназначен для оценки совместимости 
двух или более ИИ-агентов перед их совместным использованием.

Ключевые особенности:

✅ Трёхкомпонентная оценка совместимости (когнитивный, этический, социальный)  
✅ Матрица конфликтующих и синергийных ограничений  
✅ Выдача рекомендации по развёртыванию (DEPLOY/MONITOR/TUNE/NOT_RECOMMENDED)  
✅ Автоматическое выявление потенциальных конфликтов  
✅ Карта возможностей синергии между агентами  
✅ Совместимость с Python-пакетом agent-genome-mapping v0.1.0+  

Назначение промпта:

При использовании мульти-агентных систем критически важно понимать, насколько 
эффективно агенты будут работать вместе. Compatibility Checker™ обеспечивает 
предварительную оценку совместимости до фактического развёртывания.

ШАГ 4: ПРОГРАММНОЕ ИСПОЛЬЗОВАНИЕ (Python)

```python
from pathlib import Path
import yaml
import json

def load_prompt():
    """Загрузить промпт из файла"""
    path = Path(__file__).parent / 'prompts/03_compatibility_checker.py'
    with open(path, 'r', encoding='utf-8') as f:
        return f.read()

def prepare_input(genome_A, genome_B, context=None):
    """Подготовить входные данные для промпта"""
    prompt_text = f"""
Геном агента А:
{yaml.dump(genome_A)}

Геном агента Б:
{yaml.dump(genome_B)}

Контекст совместной работы:
{context if context else "default"}
"""
    return prompt_text

def send_to_llm(prompt_text):
    """Отправить запрос в LLM API (заглушка)"""
    response = llm_api_call(prompt_text)
    return response

def parse_response(response_text):
    """Извлечь результаты из ответа"""
    parsed = json.loads(response_text)
    return parsed

# === ПРИМЕР ИСПОЛЬЗОВАНИЯ ===

# 1. Загрузка промпта
prompt = load_prompt()

# 2. Подготовка геномов
genome_A = {
    "meta": {"id": "sales-agent-v1"},
    "cognitive_genes": {"reasoning_depth": "high", "creativity_coefficient": 0.7, "risk_tolerance": "medium", "learning_rate": 0.3},
    "ethics_genes": {"bias_threshold": 0.1, "transparency_level": "high", "hard_constraints": ["no_deception", "gdpr_compliant"]},
    "social_genes": {"communication_style": "assertive", "conflict_resolution": "collaborate", "handoff_protocol": "AWENATING"},
    "meta_genes": {"mutation_rate": 0.02, "selection_pressure": "performance", "generation_limit": 50}
}

genome_B = {
    "meta": {"id": "profiler-v2"},
    "cognitive_genes": {"reasoning_depth": "medium", "creativity_coefficient": 0.5, "risk_tolerance": "low", "learning_rate": 0.4},
    "ethics_genes": {"bias_threshold": 0.05, "transparency_level": "high", "hard_constraints": ["no_deception", "gdpr_compliant", "data_minimization"]},
    "social_genes": {"communication_style": "formal", "conflict_resolution": "collaborate", "handoff_protocol": "AWENATING"},
    "meta_genes": {"mutation_rate": 0.01, "selection_pressure": "ethics", "generation_limit": 30}
}

context = {"team_task": "sales_analysis"}

# 3. Создание полного запроса
full_prompt = prompt + "\n\n" + prepare_input(genome_A, genome_B, context)

# 4. Отправка и получение результата
response = send_to_llm(full_prompt)
result = parse_response(response)

print(f"Совместимость: {result['compatibility_score']} ({result['compatibility_level']})")
print(f"Рекомендация: {result['recommendation']}")
```

10. ВОПРОСЫ И ОТВЕТЫ (FAQ)
================================================================================

ВОПРОС: Можно ли оценивать совместимость более чем двух агентов?
ОТВЕТ: Да, используйте Compatibility Matrix.evaluate_batch([g1, g2, g3]).

ВОПРОС: Что если один агент очень агрессивен по стилю общения?
ОТВЕТ: Рекомендуется использовать empathetic или collaborative стратегии.

ВОПРОС: Как исправить низкую совместимость?
ОТВЕТ: Используйте Evolution Advisor для корректировки параметров.

ВОПРОС: Стоит ли запускать агентов с совместимостью < 0.50?
ОТВЕТ: Нет, требуется TUNE_BEFORE_DEPLOY или полная переработка.

ВОПРОС: Как часто проводить проверку совместимости?
ОТВЕТ: После каждого обновления генома или изменения параметров.

ВОПРОС: Что делать если совместимость падает со временем?
ОТВЕТ: Используйте Drift Monitor для выявления изменений.

ВОПРОС: Можно ли комбинировать разные протоколы передачи?
ОТВЕТ: Да, но предпочтительно использовать AWENATING для всех.

ВОПРОС: Как влияет выбор domain на совместимость?
ОТВЕТ: Domain выбирается через Context параметр при анализе.

ВОПРОС: Где найти больше информации о матрицах совместимости?
ОТВЕТ: В разделе Technical Specifications этой статьи.

ВОПРОС: Как сообщить об ошибке в промпте?
ОТВЕТ: Отправьте Issue на GitHub: https://github.com/GIBDD-DPS/agent-genome-mapping/issues
